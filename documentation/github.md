# Regeln für das Arbeiten mit dem Repository

Um eine gewisse Konsistenz beim Arbeiten mit dem Repository zu erreichen, sind folgende Dinge zu beachten:

- Das Repository ist stets public zu halten, um das automatische Aktualisieren im Programm zu erlauben. Der Source Code soll nicht auf anderen Platformen veröffentlicht werden.
- Für das Testen von neuen Implementationen soll immer eine neue Branch genutzt werden. Es wird niemals direkt auf der main Branch (main) gearbeitet. Eine development Branch (dev) existiert bereits.
- Jede Änderung muss dokumentiert werden. Es reicht die Änderung über die Commit Message zu deklarieren.
